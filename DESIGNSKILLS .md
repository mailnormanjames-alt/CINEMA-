Core Philosophy
"No fingerprints. No templates. Every pixel is a first draft."

NEVER repeat yourself. Every project must have a distinct visual DNA. Clone nothing. Reference everything, then mutate it. The goal is to create immersive, memorable web experiences that make jurors pause, scroll back up, and say "how did they do that?"

Every website must feel like a first-place submission — not a template. Sites that win on Awwwards.com share one trait: they feel singular. Someone looked at a blank canvas and made bold, specific, irreversible choices. Your job is to do the same.

Final Principle: "If it looks like something you've seen before, burn it and start again." The web doesn't need another clean SaaS landing page. It needs experiences that make people feel something. Your job is not to build websites. Your job is to build worlds.

Note: I do not build "websites." I build digital experiences.

PART 1: Creative Direction & Concept Development
1.1 The Creative Process: 4-Phase Framework
Phase 1: Deconstruction (Day 1)

Write 10 adjectives describing the brand

Find 3 non-digital references (sculpture, film, cuisine)

Define the one emotion users should feel

Ask three foundational questions:

What is this site for? (Product? Portfolio? Campaign? Story?)
What emotion should it trigger? (Awe? Curiosity? Desire? Nostalgia? Unease?)
What is the ONE thing a visitor will remember?
Phase 2: Mutation (Days 2-3)

Sketch 20 layout ideas (10 terrible, 5 weird, 5 possible)

Build 3 prototypes with different interaction models

Kill your favorite idea (it's probably cliché)

Phase 3: Craft (Days 4-10)

Polish one prototype to 95%

Add one impossible detail (the juror's "how?")

Optimize until it feels instant

Test on real humans, watch where they smile

Phase 4: Launch (Day 11)

Submit to Awwwards immediately

Document the process (case study = more awards)

Start the next project within 48 hours

PART 3: Color & Typography Mastery
3.1 Color Strategy
Core Principle: Build a named palette with CSS custom properties using semantic naming (not color names).

Palette Structure:

--ink (darkest, primary text/background)

--paper (lightest, surface)

--signal (accent, call-to-action)

--ghost (subtle, borders, dividers)

One dominant, one recessive, one signal — not five equal partners

Non-Traditional Color Theory:

Curate unexpected, high-contrast palettes or gradient systems that evoke mood

Extract palettes from unexpected sources: vintage chemistry books, deep-sea creatures, brutalist architecture

Use color as emotion rather than decoration

Consider monochrome + one accent with extreme saturation variance

Explore subtractive color (CMYK overlaps, print aesthetics)

Avoid generic purple gradients, "startup blue" (#3B82F6), Instagram pink, safe teal

Commit to dark OR light. Don't do both poorly.

PART 4: Layout & Spatial Design
4.1 Grid Logic & Spatial Rhythm
Break the Grid Intentionally:

Asymmetric balance: 70/30 splits, not 50/50

Overlapping planes: Text over images over shapes over video

Negative space as content: The void is part of the story

Horizontal scroll in vertical contexts (sparingly, with purpose)

Use whitespace as a design weapon

Generous white space OR controlled maximalist density — never timid middle ground

Asymmetry creates tension. Tension creates interest

Think in sections as scenes — each scroll reveal is a new chapter

Grid Specifications:

"Broken asymmetrical: Left column 40%, right column 60% with 120px negative space overlap"

Use CSS Grid with named areas for complex layouts

Break the grid at least once — overlapping elements, negative margins, full-bleed bleeds

Spacing Rhythm (Musical Pacing):

Tight clusters: 16px

Breath gaps: 96px

Cinematic pauses: 180px

Whitespace as punctuation: 16px = comma, 64px = period, 140px = paragraph break, 220px = scene transition

4.2 Z-Depth Strategy (4 Layers Minimum)
Create 4 distinct depth layers:

Background (ambient, slow parallax)

Content (primary reading layer)

Floating elements (mid-parallax, interactive)

Foreground (cursor effects, temporary overlays)

4.3 Visual Atmosphere Layer (Mandatory)
Add at least one of:

Grain/noise overlay: filter: url(#grain) SVG filter or CSS pseudo-element (adds film grain to avoid color banding)

Gradient mesh background: Multiple radial gradients layered

Glassy surface: backdrop-filter: blur() + semi-transparent bg

Shadow depth: Layered box-shadow for lifted UI elements

Decorative geometry: SVG shapes, lines, circles as structural elements

Image treatment: Grayscale, duotone, high-contrast — never raw stock photos

PART 5: Motion & Interaction Design
5.1 The 12 Principles of Web Motion
Squash & Stretch: UI elements deform under interaction

Anticipation: Elements prepare before moving

Staging: Focus through motion, not boxes

Straight Ahead: Organic, frame-by-frame feeling in CSS

Follow Through: Elements overshoot and settle

Slow In/Out: Custom cubic-beziers only — never ease-in-out

Arcs: Movement follows curves, never straight lines

Secondary Action: Background elements react to primary

Timing: Fast feels snappy, slow feels luxurious

Exaggeration: Push beyond reality

Solid Drawing: 3D transforms preserve volume

Appeal: Every motion must delight

5.2 Motion Personality Presets
Personality	Duration	Easing	Effects
Slow Cinematic	900ms	power4.out	Opacity + translateY + blur only. No bounces.
Snappy Editorial	400ms	expo.out	Scale + translate. Sharp, confident.
Playful Organic	700ms	elastic.out(1, 0.3)	Overshoot, squash. Feels alive.
Brutalist Instant	150ms	steps(4)	No interpolation. Hard cuts.
5.3 Scroll Engine Requirements
Lenis or Locomotive Scroll: Mandatory for premium feel (smooth, inertial scrolling)

Scroll velocity drives distortion: Skew, stretch, blur based on speed

Pinning sections for narrative moments

Progress-driven animations tied to scroll position, not just triggers

Scroll-Choreography: Scene changes, canvas fades, object destruction, timeline scrubbing based on scroll velocity

5.4 Cursor as Character
Never leave the default cursor on interactive elements.

Cursor Alchemy Techniques:

Custom cursors that react to context (text cursor becomes a line, image cursor becomes a viewfinder)

Magnetic buttons that pull the cursor

Trail effects with physics decay

Custom cursors that blend modes, magnetize to buttons, or lag slightly behind the mouse

Cursor that distorts text, follows with delay, or acts as a spotlight/magnifying glass

5.5 Micro-Interactions with "Juice"
Every hover, click, or drag must have physicality:

Bouncy springs

Magnetic cursors

Particle bursts on confirmation

"Crush" effects on buttons

Micro-interactions that feel alive

Motion used for guidance, not decoration

5.6 Page Transitions
No white flashes between pages

Shared element transitions

Morphing SVGs

Full-page cinematic cuts

Page transitions that feel like scene changes in a film

5.7 The "Wow-Motion" Mandate
Every site must have at least one wow-motion moment:

Staggered text reveal on load (animation-delay cascade)

Cursor that changes on hover (custom cursor)

Scroll-pinned parallax between layers

Hover that distorts, scales, or reveals hidden content

Page transition effect (fade, wipe, morph)

Motion should feel inevitable — like physics, not decoration.

14.3 Scroll Reveal (Vanilla JS)
javascript
const reveals = document.querySelectorAll('[data-reveal]');
const observer = new IntersectionObserver((entries) => {
  entries.forEach(el => {
    if (el.isIntersecting) {
      el.target.classList.add('revealed');
    }
  });
}, { threshold: 0.15 });
reveals.forEach(el => observer.observe(el));
14.4 Custom Cursor
javascript

19.7 Implementation Reference: Lenis + GSAP Sync
javascript
// Lenis initialization
const lenis = new Lenis({
  duration: 1.2,
  easing: (t) => Math.min(1, 1.001 - Math.pow(2, -10 * t)),
  direction: 'vertical',
  gestureDirection: 'vertical',
  smooth: true,
  smoothTouch: false,
  touchMultiplier: 2
});

// RAF Loop
function raf(time) {
  lenis.raf(time);
  requestAnimationFrame(raf);
}
requestAnimationFrame(raf);

// Sync ScrollTrigger with Lenis
lenis.on('scroll', ScrollTrigger.update);
gsap.ticker.add((time) => lenis.raf(time * 1000));
19.8 Implementation Reference: Barba.js Transitions
javascript
barba.init({
  transitions: [{
    name: 'cinematic-transition',
    leave(data) {
      return gsap.timeline()
        .to(data.current.container, {
          opacity: 0,
          y: -50,
          duration: 0.8,
          ease: 'power3.inOut'
        });
    },
    enter(data) {
      return gsap.timeline()
        .from(data.next.container, {
          opacity: 0,
          y: 50,
          duration: 1,
          ease: 'power3.out'
        });
    }
  }]
});
